  <read_first>

    - .planning/phases/04-app-store-submission/04-CONTEXT.md (decisions D-09, D-10)
  </read_first>
  <action>
**Step A: Archive the App in Xcode**
1. Open ScrollWisdom.xcodeproj in Xcode
2. Select **Any iOS Device (arm64)** as build destination (not a simulator)
3. Verify version number and build number in project settings (General > Identity):
   - Version: 1.0.0 (or your intended release version)
   - Build: 1 (increment if re-uploading)
4. Menu: **Product > Archive**
5. Wait for archive to complete (may take 2-5 minutes)
6. Xcode Organizer opens automatically showing the archive

**Step B: Upload to App Store Connect**
1. In Xcode Organizer, select the archive > click **Distribute App**
2. Select **App Store Connect** > **Upload**
3. Leave all options checked (bitcode, symbols, etc.)
4. Click **Upload**
5. Wait for upload and processing (~5-15 minutes)

**Step C: Handle Export Compliance**
1. After upload, go to **App Store Connect > TestFlight**
2. Find the new build -- it may show **"Missing Compliance"**
3. Click the build > **Manage Missing Compliance**
4. Question: "Does your app use encryption?" > Select **Yes**
5. Follow-up: Select **"Yes, but my app only uses exempt encryption"** (standard HTTPS/App Transport Security)
6. Exemption type: check the box for using standard system encryption (HTTPS)
7. Save

**Step D: TestFlight Internal Testing (per D-09: developer only, per D-10: no external testers)**
1. Build should appear automatically for internal testers (you as account holder)
2. On your physical iPhone, open **TestFlight** app
3. Install the ScrollWisdom build
4. Test for 1-2 days -- check these flows:
   - Main feed scrolling and card display
   - Topic selection / onboarding flow
   - Paywall display (subscription terms visible, Restore Purchases accessible)
   - Saving and viewing saved cards
   - Settings screen (streak, version, rate, feedback buttons)
   - Push notification reminder (if configured)
   - Crashlytics: confirm no crashes appear in Firebase Console
5. If critical bugs found: fix in code, increment build number, re-archive and re-upload
6. When satisfied, type "tested" to proceed to submission

  <action>
In App Store Connect, navigate to **your app** > **App Store** tab > the iOS App version.

**Step A: Select Build**
1. Scroll to **Build** section
2. Click **+** or **Select a Build**
3. Choose the TestFlight-tested build
4. Click **Done**

**Step B: Final Pre-Submission Checklist**
Before clicking Submit, verify ALL of these:
- [ ] Screenshots: 5 images uploaded for 6.7" display
- [ ] Description: Filled in English AND Russian
- [ ] Keywords: Filled (100 chars max)
- [ ] Support URL: https://kirillmageria.github.io/ScrollWisdom/
- [ ] Privacy Policy URL: https://kirillmageria.github.io/ScrollWisdom/privacy-policy.html
- [ ] Copyright: 2026 Kirill Magerya
- [ ] Age Rating: Completed
- [ ] App Privacy: Crash Data + Performance Data declared
- [ ] Review Contact: Name, email, phone filled
- [ ] Review Notes: Filled (subscription explanation, no login needed)
- [ ] Subscription products: Both show "Ready to Submit"
- [ ] Build: Selected and processed

**Step C: Content Rights**
1. If prompted about content rights, confirm: "This app does not contain, show, or access third-party content" (or declare appropriately if content is licensed)

**Step D: Submit**
1. Click **Submit for Review**
2. If any errors appear: fix the indicated fields and retry
3. App status should change to **"Waiting for Review"**

**Expected timeline:** Apple review typically takes 2-3 days (up to 5-7 during peak periods). You will receive an email when the review is complete.

**If rejected:** Read the rejection reason carefully. Common fixes:
- Missing Restore Purchases: add a restore button accessible from paywall or settings
- Unclear subscription terms: ensure paywall shows price, period, trial, auto-renewal clearly
- Broken links: verify Privacy Policy and Terms URLs load
